# Add a Pivot Sheet to the Live ADVANCE Workbook

The end-to-end recipe for "build a new sheet, color it, position it next to its source, push it to OneDrive". Each step has a footgun documented in code or in the wild.

---

## 0. Decide: derived vs authoritative

This skill is for **derived** sheets — sheets whose contents can be regenerated by re-running a tool. If you're adding a sheet that holds authoritative data (manual edits, source-of-truth columns), follow the export-pipeline + `PRESERVE_PC_COLS` pattern instead. Derived sheets are safe to delete-and-recreate every run; authoritative sheets are not.

`sent_log` is authoritative (rows accumulate over time, dedupe by message_id). `template_matrix` is derived (rebuild from scratch each run).

---

## 1. Build the sheet

Open the live workbook with `load_workbook(path)` (NOT `read_only=True` — you need to write). Drop any prior copy of the sheet to avoid stale headers:

```python
from openpyxl import load_workbook
wb = load_workbook(workbook_path)
if SHEET_NAME in wb.sheetnames:
    del wb[SHEET_NAME]
ws = wb.create_sheet(SHEET_NAME)
ws.append(columns)
for r in rows:
    ws.append(r)
```

### Family-grouped columns

When the sheet pivots by some categorical key (template family, intervention arm, etc.), emit a fixed block of sub-cols per category so cells are predictable. Example from `template_matrix`: 6 sub-cols per family — `<fam>_template / _due / _created / _created_source / _sent / _sent_source`.

## 2. Order columns chronologically (not alphabetically)

Alphabetical column order surfaces things in a confusing sequence. Order columns by domain semantics. Define an explicit `_FAMILY_ORDER` list in code, sort by it, and append unmapped families alphabetically at the end:

```python
_FAMILY_ORDER = ["part_int_reinvite", "part_int_reminder", "registration_confirmation", ...]

ordered = [f for f in _FAMILY_ORDER if f in seen_families]
extras = sorted(seen_families - set(_FAMILY_ORDER))
fam_list = ordered + extras
```

Test that the order is preserved (regression test: `assert seen.index("part_int_reinvite") < seen.index("registration_confirmation")`).

## 3. Apply HSL-rotation gradient coloring

Linear red→green gradient produces near-indistinguishable mid-tones. Full HSL hue rotation (0° → 300°) at fixed lightness 0.50 + saturation 0.85 gives visually distinct colors across 10–20 categories:

```python
import colorsys
from openpyxl.styles import PatternFill, Font, Alignment

def _gradient_hex(position: int, total: int) -> str:
    if total <= 1:
        return "00B050"
    p = position / (total - 1)            # 0.0 → 1.0
    hue = (p * 300.0) / 360.0             # 0° red → 300° magenta
    r, g, b = colorsys.hls_to_rgb(hue, 0.50, 0.85)
    return f"{int(round(r*255)):02X}{int(round(g*255)):02X}{int(round(b*255)):02X}"

def _lighten_hex(hex_str: str, factor: float = 0.78) -> str:
    """Mix with white. Used for body cells (factor 0.78) vs headers (0.0)."""
    r = int(hex_str[0:2], 16); g = int(hex_str[2:4], 16); b = int(hex_str[4:6], 16)
    r = int(round(r + (255 - r) * factor))
    g = int(round(g + (255 - g) * factor))
    b = int(round(b + (255 - b) * factor))
    return f"{r:02X}{g:02X}{b:02X}"
```

Apply saturated colors to header cells, lightened tints to body cells of the same column. Sub-cols of the same family share the hue so the grouping is visually obvious:

```python
for i, fam in enumerate(families):
    head_hex = _gradient_hex(i, len(families))
    body_hex = _lighten_hex(head_hex, 0.78)
    head_fill = PatternFill(start_color=head_hex, end_color=head_hex, fill_type="solid")
    body_fill = PatternFill(start_color=body_hex, end_color=body_hex, fill_type="solid")
    # apply to header row + every body row in this family's columns
```

## 4. Freeze panes + column widths

Long sheets with many columns are unreadable without frozen header + key-identifier columns. Default to freezing on `C2` (keep row 1 header + col A `email` + col B `name`):

```python
ws.freeze_panes = "C2"
ws.column_dimensions["A"].width = 32  # email
ws.column_dimensions["B"].width = 22  # name
```

## 5. Position the sheet next to its source

A freshly-created sheet lands at the end of the sheet tab order — usually wrong for analytical sheets that should sit next to their source data. Move it after a named anchor sheet:

```python
def move_sheets_after(wb, anchor: str, sheets_to_move: list[str]):
    """Put `sheets_to_move` immediately after `anchor` in the tab order."""
    objs = []
    for n in sheets_to_move:
        if n in wb.sheetnames:
            s = wb[n]
            wb._sheets.remove(s)
            objs.append(s)
    anchor_idx = wb.sheetnames.index(anchor)
    for offset, s in enumerate(objs, start=1):
        wb._sheets.insert(anchor_idx + offset, s)

move_sheets_after(wb, anchor="part_PC", sheets_to_move=["sent_log", "template_matrix"])
wb.save(workbook_path)
```

The `wb._sheets` direct manipulation is the canonical openpyxl way — `wb.move_sheet` exists but is finicky with insert positions.

## 6. Push to OneDrive (with the web-edit guard footgun)

```bash
bash ~/email_draft_automation/run/run_rclone_excel_push.sh
```

**The guard footgun:** the push script refuses to overwrite OneDrive if the cloud copy has been modified since the last `rclone_excel_sync.sh` snapshot — even if the cloud edit was just our own previous push (Excel Online reflows the ModTime). Symptom: `"OneDrive file modified since last pull (web edit detected); deferring push to next cycle."`

If you've made server-side-only edits and need to force the push:

```bash
rm -f ~/.cache/rclone-excel-pull-state    # clear the stale snapshot
bash ~/email_draft_automation/run/run_rclone_excel_push.sh
```

Only do this if you're confident no human-driven edits happened on OneDrive since the last sync. Otherwise you'll clobber their work.

## 7. Tests (mandatory before commit)

A new sheet builder needs unit tests for:

- The family ordering (chronological, not alphabetical).
- The gradient endpoints (first ≠ last, all distinct).
- Lighten arithmetic (0.0 = unchanged, 1.0 = white).
- Sheet replacement (creating doesn't leave a stale prior version).

Layer A test gate runs automatically via the PostToolUse hook after `.py` edits — don't bypass it; if it fails, the regression is real.

---

## 8. Cycle interaction

If the sheet should be rebuilt automatically on every 15-min cycle, add it to `run/automation_cycle_tasks.conf` with `state=enabled` and a sensible timeout. If it's a manual / on-demand sheet, leave it out and run with `LAYER_E_LIVE=1 .venv/bin/python3 -m tools.<sheet_builder>` on demand.

Caution: rebuilding the sheet during the cycle invalidates downstream readers — if anyone else's analysis depends on the sheet being stable, prefer manual rebuild.
